# Codex Instructions

Read `COMPOSITE.md` before making code or documentation changes.

For branch names, commit messages, formatting, tests, and pull request expectations, follow `CONTRIBUTING.md`.

When generating commit messages, follow the `Commit Messages` section in `CONTRIBUTING.md`:

- Use Conventional Commits.
- Use an imperative summary.
- Prefer scopes when they clarify the affected area.
- Keep the message concise.

When documentation files disagree, use this authority order:

1. Source code and tests.
2. The active milestone in `docs/milestones/`.
3. Standing policy documents.
4. Roadmap documents.
